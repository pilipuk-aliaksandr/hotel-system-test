# Hotel Management API

_RESTful API приложение для управления данными отелей, разработанное на Spring Boot._

## 🛠 Технологический стек

- **Java 24**

- **Spring Boot 4.0.0**

- **База данных:** H2 (с возможностью быстрой миграции на PostgreSQL или MySQL)

- **Миграции:** Liquibase

- **Документация:** Swagger (OpenAPI 3.0)

- **Сборка:** Maven


## 🚀 Быстрый запуск

### Запуск приложения

Из корневой директории выполните команду:

```
mvn spring-boot:run
```

Приложение будет запущено на **порту 8092** с контекстным путем `/property-view`.

## **Документация API**

После запуска приложения Swagger UI доступен по адресу: http://localhost:8092/property-view/swagger-ui/index.html

## 🔄 Профили баз данных

Приложение **поддерживает несколько профилей БД** для гибкого развёртывания.

### H2 (по умолчанию)

In-memory база данных для разработки и тестирования:

```
mvn spring-boot:run
```

### PostgreSQL

```
docker-compose up postgres-hotels liquibase-postgres
mvn spring-boot:run -Dspring-boot.run.profiles=postgres
```

### MySQL

```
docker-compose up mysql-hotels liquibase-mysql
mvn spring-boot:run -Dspring-boot.run.profiles=mysql
```

## 📋 Особенности реализации

* Разработана полноценная **RESTful** архитектура (контроллеры, DTO, сервисы, мапперы, доменные модели и репозитории) с соблюдением принципов **SOLID** и **Clean Architecture**.

* Спроектирована схема реляционной базы данных с настроенными связями и ограничениями. Настроены миграции **Liquibase** для обеспечения управляемой эволюции модели данных.

* Внедрен **OpenAPI Generator** и **Swagger документация (OpenAPI 3.0)** с подробными описаниями операций, реалистичными примерами запросов/ответов и четким определением опциональных полей.

* Реализована поддержка базы данных **H2** (in-memory) «из коробки» для быстрой автоматизированной проверки. Дополнительно подготовлена инфраструктура **Docker-Compose** для быстрой интеграции с **PostgreSQL** или **MySQL**.

* Реализованы профили для нескольких БД (H2, PostgreSQL, MySQL) для быстрого переключения между базами данных без изменения кода.

* **Реализован продвинутый механизм обработки исключений (Global Exception Handler):**

* * **Многоуровневое логирование:** Кастомные исключения содержат собственные уровни логирования `SLF4J`, что позволяет обработчику автоматически переключаться между уровнями `ERROR`, `DEBUG` или `INFO` в зависимости от контекста конкретной ошибки.

* * **Унифицированный формат ответов:** Гарантирована единообразная структура `ExceptionDto` для клиента для всех типов ошибок, включая ошибки валидации (`MethodArgumentNotValidException`) и непредвиденные сбои на стороне сервера.

* * **Контекстно-зависимые исключения:** Базовая абстракция (`BaseApplicationException`) позволяет фиксировать динамические параметры, коды ошибок и метаданные, которые напрямую маппятся в API-ответы с помощью MapStruct.

* Реализовано **AOP-логирование** для всех эндпоинтов контроллеров и операций с базой данных с использованием Spring AOP, обеспечивающее отслеживание запросов с HTTP-методом, URL и потоком выполнения.

* Написаны комплексные **интеграционные тесты** для REST-эндпоинтов, проверяющие бизнес-логику, коды состояний и структуру JSON-ответов.


## 📊 Эндпоинты

**POST** `/property-view/hotels` — Создать новый отель

**GET** `/property-view/hotels` — Получить список всех отелей

**GET** `/property-view/search` — Поиск отелей по параметрам

**GET** `/property-view/hotels/{id}` — Получить подробную информацию об отеле по ID

**POST** `/property-view/hotels/{id}/amenities` — Добавить удобства (amenities) к отелю

**GET** `/property-view/histogram/{param}` — Получить статистику (гистограмму) по отелям
